# 🦝 너굴 코딩 플랫폼 - 유즈케이스 및 테스트 시나리오

## Part 1: NookPhone IDE 유즈케이스

### UC-11: Python 코드 작성 및 실행
**액터:** 사용자
**목적:** IDE에서 Python 코드를 작성하고 실행하여 결과를 확인

**선행조건:**
- StoryPage를 완료하고 IDE 페이지에 도착
- Pyodide가 정상 로드됨

**기본 흐름:**
1. 사용자가 IDE 페이지 로드
2. 코드 에디터가 표시됨 (기본 코드: `print("Hello, World!")`)
3. 사용자가 새로운 Python 코드 입력
   - 예: `x = 10; y = 20; print(x + y)`
4. "실행" 버튼 클릭
5. 코드가 Pyodide에서 실행됨
6. 결과가 출력 영역에 표시됨
7. 사용자가 결과 확인

**테스트 케이스:**
- TC-11-1: 기본 print 문 실행
- TC-11-2: 변수 선언 및 계산
- TC-11-3: 반복문 (for loop) 실행
- TC-11-4: 함수 정의 및 호출
- TC-11-5: 오류 발생 시 에러 메시지 표시

**예상 결과:**
- 코드가 Pyodide WebAssembly 엔진에서 실행
- 출력이 즉시 콘솔에 표시
- 오류는 에러 박스에 명확하게 표시

### UC-12: 미션 선택 및 제출
**액터:** 사용자
**목적:** 사전 정의된 미션을 선택하여 코드를 작성하고 제출

**선행조건:**
- IDE 페이지 로드
- 미션 탭 활성화

**기본 흐름:**
1. 사용자가 미션 탭 클릭
2. 6개의 미션 목록 표시:
   - 변수와 출력 (500pts)
   - 숫자 계산 (600pts)
   - if 조건문 (700pts)
   - for 반복문 (700pts)
   - 리스트 (700pts)
   - 함수 (800pts)
3. 사용자가 특정 미션 선택 (예: "변수와 출력")
4. 미션 설명 표시
5. 에디터 탭으로 자동 전환
6. 사용자가 코드 작성
7. "제출" 버튼 클릭
8. 자동 채점 실행
9. 결과 표시 (성공/실패)
10. 성공 시 포인트 획득

**테스트 케이스:**
- TC-12-1: 미션 선택 및 설명 로드
- TC-12-2: 정답 코드 제출 → 성공
- TC-12-3: 오답 코드 제출 → 실패
- TC-12-4: 포인트 증가 확인
- TC-12-5: 다른 미션 선택 가능

**예상 결과:**
- 모든 미션이 표시되고 선택 가능
- 채점 로직이 정상 작동
- 포인트 시스템이 정상 누적

### UC-13: 진행도 추적
**액터:** 사용자
**목적:** 학습 진행 상황을 확인

**선행조건:**
- IDE 페이지 로드
- 최소 1개 미션 완료

**기본 흐름:**
1. 사용자가 진행도 탭 클릭
2. 다음 정보 표시:
   - 총 포인트: X pt
   - 완료한 미션 개수: X / 6
   - 진행률: X%
   - 획득한 뱃지: X개
3. 완료한 미션 목록 표시
4. 진행도 바 표시 (시각화)

**테스트 케이스:**
- TC-13-1: 진행도 탭 렌더링
- TC-13-2: 포인트 정확성
- TC-13-3: 미션 완료 목록 정확성
- TC-13-4: 진행도 계산 정확성
- TC-13-5: 뱃지 시스템 작동

**예상 결과:**
- 진행도가 정확하게 계산되고 표시됨
- 포인트가 정확하게 누적됨
- 진행률이 시각적으로 명확함

### UC-15: 너굴(Nook) 캐릭터 반응
**액터:** 사용자
**목적:** 너굴 캐릭터가 사용자의 행동에 반응

**선행조건:**
- IDE 페이지 로드됨

**기본 흐름:**
1. 코드 실행 성공 → 너굴이 긍정적 메시지 출력
   - 예: "좋아! 미션을 성공했어! 🎉"
2. 코드 실행 실패 → 너굴이 격려 메시지 출력
   - 예: "오류가 발생했어요. 다시 시도해봐! 🦝"
3. 미션 제출 성공 → 너굴이 축하 메시지
4. 미션 제출 실패 → 너굴이 응원 메시지

**테스트 케이스:**
- TC-15-1: 성공 시 메시지 (다양성 확인)
- TC-15-2: 실패 시 메시지 (다양성 확인)
- TC-15-3: 메시지 변경 확인
- TC-15-4: 너굴 애니메이션 (호흡 애니메이션)

**예상 결과:**
- 너굴이 상황에 맞는 메시지 표시
- 최소 5개 이상의 다양한 메시지 (게임 재미성)
- 애니메이션이 부드럽게 작동

## Part 3: 통합 테스트 시나리오

### Scenario-1: 전체 사용자 여정
**시나리오:** 신규 사용자가 처음부터 끝까지 학습

```
1. http://localhost:3002 방문
2. EntryPage 로드 확인
3. 클릭하여 StoryPage로 진행
4. 3개 씬 모두 진행
   - 텍스트 타이핑 확인
   - 오디오 재생 확인
5. "시작하기" 클릭 → IDE 페이지로 이동
6. IDE 페이지 로드
   - 미션 탭 확인
   - 에디터 탭 확인
   - 진행도 탭 확인
7. 첫 번째 미션 선택
8. 코드 입력 및 실행
9. 결과 확인
10. 미션 제출
11. 포인트 획득 확인
12. 진행도 탭에서 증가된 포인트 확인
```

**검증 항목:**
- ✓ 모든 페이지 로드 성공
- ✓ 라우팅 정상 작동
- ✓ 데이터 흐름 정상
- ✓ UI 요소 모두 표시
- ✓ 상호작용 모두 반응

### Scenario-2: IDE 완전 테스트
**시나리오:** IDE의 모든 기능을 순차 테스트

```
1. IDE 로드
2. 기본 코드 확인: print("Hello, World!")
3. 실행 버튼 클릭
4. 출력 확인: Hello, World!
5. 새 코드 입력:
   ```python
   numbers = [1, 2, 3, 4, 5]
   total = sum(numbers)
   print(f"합계: {total}")
   ```
6. 실행 버튼 클릭
7. 출력 확인: 합계: 15
8. 미션 탭 클릭
9. 첫 번째 미션 선택
10. 에디터로 이동
11. 정답 코드 작성
12. 제출 버튼 클릭
13. 성공 메시지 확인
14. 포인트 확인
15. 진행도 탭 클릭
16. 포인트 누적 확인
17. 진행도 바 확인
```

**검증 항목:**
- ✓ Pyodide 정상 작동
- ✓ 코드 실행 결과 정확
- ✓ 미션 선택 가능
- ✓ 자동 채점 작동
- ✓ 포인트 시스템 작동
- ✓ 진행도 계산 정확
