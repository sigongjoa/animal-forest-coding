### 01_MODULAR_ASSET_SYSTEM.md (모듈형 에셋)
**내용**: 이미지를 "레고 블록"처럼 조합하는 방식

**핵심 개념**:
```
기존: 매 장면마다 완전한 일러스트 필요
└─ Episode 1-4: 21장 × 4 = 84장

제안: 초기 에셋 + 조합
└─ 초기 에셋: 배경 5 + 스프라이트 8 + UI 8 = 21장
└─ 이후 에피소드: 텍스트만 변경, 에셋 재사용
```

**기술 세부사항**:
- 배경 레이어 (5가지)
- 캐릭터 스프라이트 (너굴 5표정 + 플레이어 3포즈)
- UI 요소 (대화창, 배너, 진행도 등)
- Canvas/JavaScript 기반 렌더링

**구현 코드 포함**: 
- AssetManager 클래스
- SceneComposer 클래스
- 장면 정의 JSON 예시

**읽을 대상**: 프론트엔드 엔지니어, 디자이너

### 02_NEURO_NOOK_AI_SYSTEM.md (AI NPC)
**내용**: Claude API를 이용한 동적 대화 생성

**핵심 개념**:
```
기존: 정해진 에러 메시지
└─ "❌ 오류: int loan = 49800; 형태로 선언해야 합니다."
└─ 딱딱하고 교육적 가치 낮음

제안: AI 기반 맞춤형 피드백
└─ "이런구리! 변수를 선언할 때는... 생각해 봐! 넌 할 수 있어!"
└─ 자연스럽고 교육적 가치 높음
```

**기술 세부사항**:
- Claude API 통합
- System Prompt 설계
- 응답 캐싱 (히트율 60%+)
- 15가지 에러 타입별 피드백

**구현 코드 포함**:
- NookAIService 클래스
- FeedbackCache 클래스
- API 엔드포인트 설계
- 프론트엔드 통합 코드

**읽을 대상**: 백엔드 엔지니어, 교육 콘텐츠팀

## 💡 자주 묻는 질문 (FAQ)

**Q: 얼마나 복잡한 작업인가?**
A: MVP 2주, 완전 구현 6주. 대부분 구현 코드를 제공하므로 복잡도는 중간 수준입니다.

**Q: 비용이 많이 드나?**
A: 초기 개발 $51,000이지만, 연간 운영 $4,200은 매우 저렴합니다. 1년 후 ROI 달성 가능.

**Q: AI 응답이 항상 정확한가?**
A: 교육 목적이므로 완벽함이 아닌 "맞춤형 피드백"이 목표입니다. 캐싱으로 검증된 응답만 사용.

**Q: 기존 Episode 1에 영향이 있나?**
A: 없습니다. 마이그레이션이 투명하게 이루어지며, 모든 기능은 동일합니다.

**Q: Episode 2는 언제 나오나?**
A: MVP 완성 후 2주 내 완성. 기존 에셋 재사용으로 매우 빠릅니다.
